# AWS SAA-C03 模擬考系統

一個純前端、免安裝的 **AWS Solutions Architect Associate (SAA-C03)** 模擬考試網頁應用程式，支援繁體中文與英文介面。

---

## 快速開始

```
直接用瀏覽器開啟 index.html，無需任何伺服器或安裝步驟。
```

> **相依性**：`index.html` 與 `questions.js` 必須在同一目錄下。

---

## 功能總覽

### 🎯 考試設定
| 設定項目 | 可選範圍 |
|---------|---------|
| 題目數量 | 15 / 30 / 65（預設）/ 100 / 全部題目 |
| 考試時間 | 15 / 30 / 60 / 90 / 130 分鐘（官方）/ 不限時 |
| ESL 延長 | 有時間限制時可勾選，延長時間與題目數量等比例（65 題對應 +30 分鐘） |

### 📝 答題功能
- **單選 / 多選題**支援，多選題自動提示需選幾個答案
- **❓ 標記為不確定**：答題時可對任一題標記不確定，在題目總覽和答題記錄中醒目顯示
- **題目總覽側欄**：顯示所有題目狀態（已答、未答、當前、不確定），可快速跳題
- **鍵盤快捷鍵**：← / → 切換題目，`A`–`F` 直接選答案
- **計時器**：剩餘 5 分鐘時閃爍警示，時間到自動交卷
- **交卷按鈕**：不限時模式顯示「交卷」，有時間限制時顯示「提前交卷」

### 🏆 AWS 計分結構模擬

本系統模擬 AWS SAA-C03 正式考試的 **50 計分題 + 15 不計分題（Pretest）** 結構：

| 題數模式 | 計分題 | 不計分題 |
|---------|--------|---------|
| 15 題 | 12 題 | 3 題 |
| 30 題 | 23 題 | 7 題 |
| 65 題（標準） | 50 題 | 15 題 |
| 100 題 | 77 題 | 23 題 |

- 不計分題由系統隨機指定，考試過程中與一般題目無區別
- 最終得分 = 答對計分題數 ÷ 計分題總數
- 答題記錄中以「不計分」標籤標示

### 📊 成績報告
- 顯示得分、通過 / 未通過結果
- **練習安全目標對照色帶**：以漸層色帶標示四個目標區間

  | 區間 | 含義 |
  |------|------|
  | 🔴 0 – 72% | 未通過 |
  | 🟡 72 – 80% | 及格，但建議繼續提升 |
  | 🟢 80 – 85% | 達到安全練習目標 |
  | 🟢 85%+ | 準備充分，信心出發 |

- **ℹ️ AWS 計分說明**：說明縮放計分制、題目加權、無猜題懲罰、試驗題機制

### 📖 答題記錄與解析
- 每題顯示正確答案、你的選擇及解析說明
- 答案即時對照最新題庫（手動修正 `questions.js` 答案後，舊記錄自動反映新答案）
- **篩選器**：全部 / 答錯 / 答對 / 未作答 / ❓ 不確定 / ➖ 無標準答案
- **📝 筆記功能**：每道題目可新增/編輯/刪除個人筆記，以題目 ID 為鍵永久儲存於 localStorage，跨考試共用
- **🔖 書籤功能**：每場考試可標記一道書籤題目；畫面右下角的圓形書籤鈕會在書籤題目捲出畫面後自動顯示，點擊即可跳回

### 📈 首頁統計數據
統計數據**僅計入與正式考試比例相符的紀錄**（每題 2 分鐘且計分比例與 65 題標準相同），避免練習模式影響統計參考性。

### 💾 資料儲存（localStorage）
| 資料 | Key |
|------|-----|
| 考試歷史紀錄 | `aws_saa_history` |
| 個人筆記 | `aws_saa_notes` |
| 書籤 | `aws_saa_bookmarks` |
| 語言偏好 | `aws_lang` |
| 深色模式偏好 | `aws_dark` |

### 📋 歷史紀錄
- 儲存所有考試成績，顯示日期、時間、題數、分數、通過/未通過
- 可查看任一場考試的完整答題記錄
- 支援刪除單筆紀錄
- **⬇️ 匯出 Markdown**：每場考試可匯出完整成績單（含答案、解析、筆記）

### 🌐 其他
- **深色模式 / 淺色模式** 切換；首次訪問時自動跟隨系統設定（`prefers-color-scheme`）
- **繁體中文 / 英文** 介面切換（動態重繪，不需重整頁面）

## 檔案結構

```
aws/
├── index.html              # 主程式（Single Page Application，純前端）
├── questions.js            # 題庫（QUESTION_BANK 常數陣列）
├── extract_pdf.py          # 步驟 1：從 PDF 提取純文字（需 PyMuPDF）
├── parse_questions.py      # 步驟 2：將純文字解析為 questions.js
├── AWS Certified Solutions
│   Architect Associate
│   SAA-C03.pdf             # 原始題庫 PDF（來源文件）
└── AWS SAA-03 Solution.txt # 解析後的純文字（中間產物）
```

## 題庫格式

`questions.js` 匯出一個名為 `QUESTION_BANK` 的 JavaScript 陣列，每個物件結構如下：

```js
{
  "id": 1,                          // 題目編號
  "question": "題目內容...",
  "options": {
    "A": "選項 A 文字",
    "B": "選項 B 文字",
    "C": "選項 C 文字",
    "D": "選項 D 文字"
  },
  "correctAnswers": ["A"],          // 正確答案（多選題為多個字母）
  "numChoices": 1,                  // 需選幾個答案（多選題 > 1）
  "explanation": "解析說明..."      // 可選，留空字串表示無解析
}
```

> 無答案（`correctAnswers: []`）或選項不足 2 個的題目，在系統初始化時會自動從題庫移除，不會出現在考試中。

## 從 PDF 重建題庫

若需要從原始 PDF 重新產生題庫，依序執行以下步驟：

### 環境需求

```bash
pip install pymupdf
```

### 步驟

```bash
# 步驟 1：提取 PDF 文字
python extract_pdf.py

# 步驟 2：解析為 questions.js
python parse_questions.py
```

`parse_questions.py` 會讀取步驟 1 產生的 `*_extracted.txt`，輸出可直接被 `index.html` 引用的 `questions.js`。

## AWS 計分機制說明

> 本系統以「答對計分題數 ÷ 計分題總數」計算百分比，與正式考試的縮放分數存在差異。

| 項目 | 說明 |
|------|------|
| 計分制度 | 縮放計分（Scaled Score），範圍 100–1000 |
| 通過分數 | **720 分**（約等於 72%） |
| 題目加權 | 不同題目具有不同分值 |
| 猜題懲罰 | **無**，答錯不扣分，建議所有題目都作答 |
| 試驗題 | 每場含若干不計分的實驗性題目（Pretest），考生無法辨識 |
| 業界建議 | 模擬考穩定達到 **80–85%+** 再報考，保留足夠分數緩衝 |
